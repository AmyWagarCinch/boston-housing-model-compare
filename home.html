Machine Learning to Predict Home Values

This is a Machine Learning project done to compare different models using basic Linear Regression and Random forest Machine Learning concepts. The dataset was collected in 1978 with 506 entries and 13 Variables. Each variable represents different features for homes in various suburbs in Boston, Massachusetts. Here is a list of all the features in the original dataset:

  • CRIM - per capita crime rate by town
  • ZN - proportion of residential land zoned for lots over 25,000 sq. ft.
  • INDUS - proportion of non-retail business acres per town.
  • CHAS - Charles River dummy variable (1 if tract bounds river; 0 otherwise)
  • NOX - nitric oxides concentration (parts per 10 million)
  • RM - average number of rooms per dwelling
  • AGE - proportion of owner-occupied units built prior to 1940
  • DIS - weighted distances to five Boston employment centers
  • RAD - index of accessibility to radial highways
  • TAX - full-value property-tax rate per $10,000
  • PTRATIO - pupil-teacher ratio by town
  • LSTAT - % lower status of the population
  • MEDV - Median value of owner-occupied homes in $1000's

In each model, we used different numbers and combinations of features to see and compare the testing accuracy outcomes. We found out that random forest regression works better with smaller number of features.

Purpose:

The purpose of this project was to first, compare the linear regression method to the random forest regression method to determine which is optimal. We discovered that the random forest regression is a better method because you can maintain higher accuracy with fewer data points than the linear aggression allows. Then, we created an app to allow users to evaluate the testing accuracy of the random forest regression using any number of factors from the dataset.
